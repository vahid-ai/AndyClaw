# Security Audit Report: AndyClaw

**Date:** 2026-03-12
**Auditor:** Claude Code
**Scope:** Full codebase security review

---

## Executive Summary

| Severity | Count | Areas |
|----------|-------|-------|
| **CRITICAL** | 6 | Code execution, TLS bypass, cleartext traffic |
| **HIGH** | 12 | Shell injection, PendingIntent mutable, WebView, logging |
| **MEDIUM** | 8 | Missing certificate pinning, unencrypted databases, permissions |
| **LOW** | 5 | JitPack dependencies, plain SharedPrefs for non-sensitive data |

---

## Critical Findings

### 1. BeanShell Arbitrary Code Execution

**File:** `app/.../skills/builtin/CodeExecutionSkill.kt:109`

```kotlin
interpreter.eval(code)  // Direct eval of LLM-supplied code
```

**Risk:** Complete device compromise if agent is manipulated. Executes arbitrary Java with full Android API access.

---

### 2. Disabled Hostname Verification

**File:** `AndyClaw/.../gateway/GatewayTls.kt:65`

```kotlin
hostnameVerifier = HostnameVerifier { _, _ -> true }
```

**Risk:** MITM attacks - accepts any certificate regardless of hostname.

---

### 3. Shell Command Injection (WiFi Password)

**File:** `app/.../skills/builtin/ConnectivitySkill.kt:196`

```kotlin
"cmd wifi connect-network \"$ssid\" wpa2 \"$password\""
```

**Risk:** Password/SSID can contain shell metacharacters allowing arbitrary command execution.

---

### 4. Cleartext HTTP in Debug Builds

**File:** `app/src/debug/res/xml/network_security_config.xml:4-9`

```xml
<domain-config cleartextTrafficPermitted="true">
  <domain includeSubdomains="true">openclaw.local</domain>
  <domain includeSubdomains="true">ts.net</domain>
</domain-config>
```

**Risk:** Debug APKs allow unencrypted traffic to local development domains.

---

### 5. Unencrypted Room Databases

**Files:** `SessionDatabase.kt`, `MemoryDatabase.kt`, `AgentTxDatabase.kt`

**Risk:** Three databases store conversations, memory, and transaction history without encryption. Accessible via device backup or physical access.

---

### 6. Backup Enabled with Empty Rules

**File:** `AndroidManifest.xml:86-88`

```xml
android:allowBackup="true"
android:fullBackupContent="@xml/backup_rules"
```

**Risk:** All app data (including encrypted SharedPrefs, databases) can be extracted via ADB backup. Backup rules file has all exclusions commented out.

---

## High Findings

### 7. FLAG_MUTABLE PendingIntents

| File | Line | Issue |
|------|------|-------|
| `TermuxCommandRunner.kt` | 155 | Termux command results can be modified |
| `AuroraStoreSkill.kt` | 481 | Install results can be manipulated |

**Fix:** Change `FLAG_MUTABLE` to `FLAG_IMMUTABLE`

---

### 8. Wallet Address Logging

**File:** `MessengerSkill.kt:248,268,297`

```kotlin
Log.i(TAG, "XMTP identity address: $identityAddress")
Log.d(TAG, "sendMessageToUser: walletSDK.getAddress() returned: '$userAddress'")
```

**Risk:** Wallet addresses logged in plaintext, visible in logcat.

---

### 9. WebView Security

**File:** `node/CanvasController.kt:121,129`

- `evaluateJavascript()` with arbitrary JS
- No explicit WebViewClient for URL validation
- Loads user-controlled URLs directly

---

### 10. TOFU Certificate Trust

**File:** `GatewayTls.kt:34-58`

```kotlin
if (params.allowTOFU) {
    onStore?.invoke(fingerprint)
    return  // Accepts cert on first use without validation
}
```

**Risk:** First connection can be intercepted before fingerprint is established.

---

### 11. Exported Services Without Manifest Permissions

| Service | Issue |
|---------|-------|
| `HeartbeatBindingService` | Exported, no `android:permission` attribute |
| `LauncherBindingService` | Exported, no `android:permission` attribute |

Runtime UID checks exist but binding succeeds before check executes.

---

### 12. AiNameProvider - No Permission Check

**File:** `AiNameProvider.kt:27-42`

Any app can query the configured AI name - no UID or permission validation implemented.

---

## Medium Findings

### 13. No Certificate Pinning

All OkHttpClient instances use default configuration without certificate pinning:

- `AnthropicClient.kt`
- `TelegramBotClient.kt`
- `ClaudeOauthTokenManager.kt`
- `ModelDownloadManager.kt`
- `OpenAiEmbeddingProvider.kt`

---

### 14. API Keys in BuildConfig

**Files:** Multiple skill files reference `BuildConfig.ALCHEMY_API`, `BuildConfig.BUNDLER_API`

**Risk:** Keys compiled into APK, extractable via reverse engineering.

---

### 15. Termux/Shell Command Execution

**Files:** `ShellSkill.kt:176-178`, `TermuxSkill.kt:136`

Basic injection detection exists but can be bypassed with advanced obfuscation. Commands run with app privileges.

---

### 16. Extensive Android Permissions

50+ permissions requested including privileged operations:

- `INSTALL_PACKAGES`
- `DELETE_PACKAGES`
- `REBOOT`
- `MODIFY_PHONE_STATE`
- `FORCE_STOP_PACKAGES`

Expected for ethOS but represents large attack surface on stock Android.

## Recommended Fixes

### Priority 1: Critical

1. Fix hostname verifier - implement proper validation in `GatewayTls.kt`
2. Escape shell parameters in `ConnectivitySkill.kt` using proper quoting
3. Set `android:allowBackup="false"` or explicitly exclude sensitive databases
4. Encrypt Room databases with SQLCipher

### Priority 2: High

5. Change `FLAG_MUTABLE` to `FLAG_IMMUTABLE` in PendingIntents
6. Remove wallet address logging from `MessengerSkill.kt`
7. Add manifest permissions to exported services
8. Add UID check to `AiNameProvider`

### Priority 3: Medium

9. Implement certificate pinning for critical API endpoints
10. Consider sandboxing BeanShell execution in isolated process

---

## Overall Assessment

**The app is functional but has security gaps typical of rapid development.**

The architecture correctly separates sensitive storage (EncryptedSharedPreferences) and includes good defensive measures (LeakDetector, SSRF protection). However, the combination of code execution capabilities, disabled TLS validation, and unencrypted databases creates risk.

### Recommendations by Deployment Target

| Target | Recommendation |
|--------|----------------|
| **Stock Android** | Address critical items 1-4 before distribution |
| **ethOS Privileged Mode** | System-level isolation provides additional protection, but fixes remain recommended |
